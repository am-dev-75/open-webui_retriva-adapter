# Retriva Constitution — Thin Adapter UX Routing

## Product law
- Users must receive immediate, meaningful feedback for ingestion directives.
- Users must receive immediate, meaningful feedback for upload-only turns.
- The chat LLM is reserved for actual questions, not ingestion control messages.

## Architecture law
- The Thin Adapter performs turn classification before forwarding.
- Synthetic acknowledgements are generated by the adapter, not by the LLM.
- Existing ingestion and retrieval semantics remain unchanged.

## Scope law
Out of scope:
- Open WebUI frontend changes
- changes to ingestion_api_v1 semantics
- changes to OpenAI-compatible answer generation for real questions
